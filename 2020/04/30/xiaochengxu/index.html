小程序的MVVM架构

model(数据) ←→ view model（桥梁） ← → view vue 的mvvm架构

小程序的 mvvm 架构 view（视图层） MINA框架 model（逻辑层）

编程范式：1、命令式编程：原生操作DOM 2、声明式编程

小程序的架构和配置

  • 小程序的很多开发需求被规定在配置文件中

  • app.json 全局配置

  • page.json页面配置

  • project.config.json 编译基础库设置文件

  • sitemap.json 配置小程序是否允许被微信搜索

    部分全局配置 详情参考微信官方文档
    属性 类型 必填 描述
    pages string[] 是 页面描述列表
    windows Object 否 全局的默认窗口表现
    tabBar Object 否 底部tab栏的表现

    所以页面必须在pages中注册 每一项对应一个路径信息 windows窗口（不是页面内容）显示指定窗口如何显示，还包含了其他属性

局部配置

在相应的页面编辑.json文件 直接设置属性，不用写 window 。页面配置会覆盖全局配置

小程序组件

text组件

基础使用： 显示文本，\n表示换行 属性：1、selectable是否可以被选中（默认长按不可选中）

2、文本空格大小 space

3、decode属性：是否解码文本

button组件

属性：1、size （按钮大小）

2、type（按钮颜色）

3、disabled(不可点击) …………

view组件

view的基本使用 一个容器 视图组件，块级元素，独占一行 放文本、其他组件

属性：hover—class 点击变换样式 hover-start-time=”300“ 按住后多久出现点击状态 hover-stop-propagation=”” 是否阻止祖先组件状态变化（案例在view嵌套时候）

image组件

重点：1、可以写成单标签和双标签 单： 双 2、默认有大小 占据位置 3、行内块级元素 相对路径/绝对路径/链接 调取相册里的图片

4、图片懒加载 lazy-load 默认false

5、监听图片加载完成 bindload

6、show-menu-by-longpress 长按识别小程序码

7、mode 缩放、剪裁模式

非常多的数值…………

input组件

input表单组件，接受用户输入 value： 显示默认文本 type ：键盘类型 password ：明文暗文 ……

input绑定事件 事件监听

scroll-view组件

局部滚动 1、水平滚动 scroll-x 2、竖直滚动 scroll-y

共同属性 wxml（组件）

属性名 类型 描述 注解
id String 组件唯一标识 整个页面唯一
class String 组件样式类 在对应的wxss中定义的样式类
style String 组件内联样式 可以动态设置的内联样式
hidden Boolean 组件是否显示 所有组件默认显示
date-* Any 自定义属性 组件上触发事件，会默认发送给事件处理函数
bind/catch EventHandler 组件的事件

样式的三种写法

wxss

1、行内样式（内联样式） /↑ 上面的共同属性 /

2、页内样式

3、全局样式 在app.wxss中编辑 行内样式优先级比较高>页面样式>全局样式
